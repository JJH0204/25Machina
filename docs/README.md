# 문서 디렉터리

이 디렉터리에는 프로젝트의 Git 워크플로우 관련 문서가 포함되어 있습니다.

## 📚 문서 목록

### 1. [현재 저장소 상태 및 작업 방법](current-repo-status.md)
**추천 시작점!** 이 저장소에 특화된 가이드입니다.
- 현재 저장소의 상태 설명
- 바로 사용 가능한 명령어 예제
- 일반적인 작업 시나리오

### 2. [Git 워크플로우 가이드](git-workflow-guide.md)
Main 브랜치를 기준으로 안전하게 작업하는 방법에 대한 완전한 가이드입니다.
- 4가지 주요 시나리오 (새 브랜치, rebase, reset, stash)
- 안전 수칙 및 백업 전략
- 복구 방법
- 문제 해결

### 3. [Git 빠른 참조](git-quick-reference.md)
자주 사용하는 Git 명령어 모음입니다.
- 카테고리별 명령어 정리
- 복사해서 바로 사용 가능
- 이모지로 쉬운 탐색

## 🎯 사용 방법

### 처음 시작하는 경우
1. [현재 저장소 상태 및 작업 방법](current-repo-status.md)부터 읽어보세요
2. 필요한 시나리오를 찾아 명령어를 복사해서 사용하세요

### 특정 시나리오가 필요한 경우
- [Git 워크플로우 가이드](git-workflow-guide.md)에서 해당 시나리오를 찾으세요
  - 시나리오 A: 새 기능 브랜치 생성
  - 시나리오 B: Rebase로 브랜치 업데이트
  - 시나리오 C: Main 브랜치 동기화
  - 시나리오 D: Stash로 임시 저장

### 빠른 명령어 검색
- [Git 빠른 참조](git-quick-reference.md)에서 필요한 명령어를 찾으세요

## ⚠️ 중요 원칙

1. **백업 우선**: 위험한 작업 전에는 항상 백업 브랜치를 생성하세요
2. **검증 후 진행**: 각 단계마다 `git status`로 상태를 확인하세요
3. **force-with-lease 사용**: 강제 푸시 시 `--force-with-lease`를 사용하세요
4. **협업 고려**: 공유 브랜치는 팀원과 사전 합의 후 작업하세요

## 🆘 도움이 필요한 경우

- 문제가 발생했다면: [Git 워크플로우 가이드](git-workflow-guide.md)의 "문제 해결" 섹션 참조
- 복구가 필요하다면: "안전 수칙 및 복구 방법" 섹션 참조
- 기본 명령어가 필요하다면: [Git 빠른 참조](git-quick-reference.md) 참조
